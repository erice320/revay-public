SedonaOffice — Migration Data Export Guide

Field-level instructions for pulling your data out of SedonaOffice before onboarding. Send these files to your SA ahead of your kickoff call.

📚 Sources used in this guide Rev.io PSA data model: Derived from live Rev.io PSA Billing API (api.psarev.io) — field names reflect actual API response schemas from confirmed endpoints. · SedonaOffice: Customer Query Builder KB (Bold Group, Sep 2024) · GL Query Builder KB (Bold Group, Jan 2023) · SedonaAPI 2.0 KB (Bold Group) · Note: Rev.io PSA service/equipment endpoints are not yet public — those sections are marked and will be importable once the API opens.

Customer & Contact Data Source: sedonaoffice.knowledgeowl.com — Customer Query Builder (Sep 2024)
1

Customer Accounts

Must Have
How to export
  1. In SedonaOffice toolbar → Query → Customer Query
  2. In the Fields to Display panel, add the fields listed in the mapping below
  3. Leave the filter frame blank (pulls all customers) or filter Status = Active
  4. Click the green arrow to run → click Export → CSV
  5. Save as sedona-customers.csv
Field mapping — SedonaOffice → Rev.io PSA
SedonaOffice FieldRev.io PSA FieldNotes
Customer_Number→accountNumberWill be re-assigned in Rev.io; keep for reference matching
Customer_Name→nameRequired
Customer_Type→identity (Business/Individual)Map "Commercial" → Business, "Residential" → Individual
Customer_Status→status (Open/Closed)Active → Open
Bill_Address1→address.addressLine1
Bill_City→address.cityMunicipality
Bill_State→address.stateProvinceCode2-letter code
Bill_Zip→address.postalCode
Bill_Email→emailAddresses[ ]Primary billing email
Parent_Customer_Number→parentCustomerIdFor multi-site accounts — link child to parent
Sales_Rep→customFields.salesPersonRev.io PSA custom field confirmed in API
2

Contacts per Customer

Must Have
How to export
  1. Query → Customer Query
  2. Add contact fields: Customer_Number, Bill_Contact_Name, Bill_Contact_Title, Bill_Contact_E_Mail, Bill_Contact_Phone, Site_Contact_Name, Site_Contact_E_Mail, Site_Contact_Phone
  3. Export → CSV as sedona-contacts.csv
Field mapping
SedonaOffice FieldRev.io PSA FieldNotes
Customer_Number→customerId (link)Used to associate contact to customer
Bill_Contact_Name→name
Bill_Contact_Title→title
Bill_Contact_E_Mail→emailAddresses[ ]
Bill_Contact_Phone→phoneNumber
Site_Contact_Name→name (separate contact row)Create as second contact; set isPrimaryCustomerContact = false
Site_Contact_E_Mail→emailAddresses[ ]
Contact_Type→contactTypeDescriptione.g., Billing, Technical, Executive
Recurring Billing & Products Source: SedonaOffice KB — Customer Query Builder; Rev.io PSA API /products schema
3

Recurring Revenue / RMR Schedules

Must Have
  1. Query → Customer Query
  2. Add: Customer_Number, Customer_Name, System_Number, System_Type, RMR_Amount, Billing_Cycle, Next_Bill_Date, Bill_Code, Bill_Code_Description
  3. Export as sedona-rmr.csv
SedonaOffice FieldRev.io PSA FieldNotes
Customer_Number→customerId (link)
RMR_Amount→product.rateMonthly billing amount per service line
Bill_Code→product.productNumberMaps to a product in Rev.io catalog
Bill_Code_Description→product.productName
Billing_Cycle→bill profile / billing frequencyMonthly, Quarterly, Annual
Next_Bill_Date→billing start dateEnsures no gap or double billing on go-live
System_Type→product.productCategoryNamee.g., Alarm Monitoring, Access Control, Video
⚠️ Each system/service line per customer will be a separate row. Do not filter these out — Rev.io needs one product subscription per billing line.
4

Bill Codes / Service Catalog

Must Have
  1. Go to Accounting → Setup → Bill Codes (or Lists → Bill Codes)
  2. Right-click the grid → Export to CSV/Excel
  3. If unavailable, use Query Builder to pull Bill_Code and Bill_Code_Description fields
  4. Save as sedona-bill-codes.csv
SedonaOffice FieldRev.io PSA FieldNotes
Bill_Code→productNumberYour internal SKU/code
Bill_Code_Description→productName
Default_Rate→rateStandard billing rate for this code
Cost→cost
GL_Account→glCodeIdLinks revenue to your chart of accounts
Taxable_YN→(tax configuration)Flag for SA during product setup
One_Time_YN→productTypeOneTimeTrue = one-time charge; False = recurring
GL Codes & Payment Configuration Source: SedonaOffice KB — GL Query Builder (Jan 2023); Rev.io PSA API /gl-codes & /payment-methods schemas
5

GL Codes / Chart of Accounts

Must Have
  1. Query → GL Query Builder
  2. Add: GL_Account_Number, GL_Account_Description, Account_Type, Active
  3. Filter: Active = Yes
  4. Export as sedona-gl-codes.csv
SedonaOffice FieldRev.io PSA FieldNotes
GL_Account_Number→glCodeId / glCodeLedgerYour account code (e.g., 4000, 4100)
GL_Account_Description→glCodeDescriptione.g., "Monitoring Revenue", "Installation Revenue"
Account_Type→(revenue vs. expense classification)Rev.io primarily uses revenue GL codes on products
💡 GL codes must be set up in Rev.io PSA before products can be fully configured.

Equipment & Service Data Source: SedonaOffice Customer Query Builder KB; Rev.io PSA service endpoints not yet public
7

Installed Systems / Equipment per Site

Helpful
  1. Query → Customer Query
  2. Expand System category and add: System_Number, System_Type, System_Location, Install_Date, Panel_Type, Monitoring_Status
  3. Include Customer_Number for linking
  4. Export as sedona-systems.csv
SedonaOffice FieldRev.io PSA (future)Notes
System_Number→asset / equipment ID
System_Type→asset type / categorye.g., Burglar Alarm, CCTV, Access Control
System_Location / Site_Address→service location / sitePhysical address where system is installed
Panel_Type→equipment model / description
Install_Date→asset install date
📌 Rev.io PSA service/equipment endpoints are not yet in the public API. Collect this data now — your SA will configure it manually during onboarding, and it will be importable once the API opens.
